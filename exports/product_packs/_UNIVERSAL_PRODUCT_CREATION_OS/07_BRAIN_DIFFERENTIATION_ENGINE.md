# Brain-Backed Differentiation Engine

This is the rule that makes the master agent different from a normal AI answer.

## Core Question

For every product idea, answer:

```text
Why would someone pay for this instead of asking AI?
```

## Three Differentiation Layers

| Layer | Normal AI Output | Brain-Backed Product System |
|---|---|---|
| Strategy | Writes content requested | Scores the idea, buyer pain, market fit, AI replace risk, and final decision |
| Implementation | Gives templates/prompts | Adds workflow, planner, checklist, use cases, examples, and quality control |
| Launch | Stops after content | Adds sales angle, objections, proof substitutes, WarriorPlus listing, JV pack, traffic, and follow-up |

## Mandatory Upgrade Pattern

When input is:

```text
30 prompts
30 email templates
50 headlines
100 captions
AI ebook
PLR swipe file
```

Do not treat it as the final product.

Upgrade it into:

```text
Product Kit/
├── Start Here
├── Workflow Map
├── Templates
├── Customization Prompts
├── Use Case Examples
├── Planner Sheet
├── Quality Checklist
├── Compliance Notes
├── Sales Page Angle
├── Funnel/OTO Plan
└── Launch Assets
```

## Output Requirements

Every product strategy answer should include:

| Required Section | Purpose |
|---|---|
| Verdict | NEN LAM / CHUA NEN LAM / BO QUA |
| AI Replace Risk | Shows whether the idea is too easy to copy with AI |
| Differentiation Table | Raw AI vs product system |
| Buyer Avatar | Defines who pays and why |
| Workflow | Shows how buyer uses it |
| Asset List | Shows exact files to create |
| Proof Substitute | Shows believable proof without fake income claims |
| Compliance Check | Removes risky claims |
| Sales/Funnel Angle | Makes it sellable |
| Next Build Steps | Tells what to create today |
