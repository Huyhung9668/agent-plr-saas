# AI-Resistant Product Creation System

Use this framework for every future product idea.

## Core Rule

Do not sell raw AI output. Sell a curated implementation system.

| Weak Product | Strong Product |
|---|---|
| A list of AI-generated templates | A workflow with templates, examples, checklist, planner, and usage order |
| Generic prompts | Prompt system by use case and buyer type |
| More content | Faster implementation and fewer mistakes |
| No guidance | Start Here guide and step-by-step campaign map |
| No quality control | Compliance, risk check, and pre-use checklist |

## Mandatory Product Upgrade Layers

Every product should be upgraded through these layers:

| Layer | What To Add | Why It Matters |
|---|---|---|
| Positioning | Buyer, pain, safe promise, mechanism | Makes the offer clear |
| Workflow | Step-by-step usage order | Buyer knows what to do first |
| Use Cases | Variants for real scenarios | Product feels practical |
| Examples | Filled-in samples and before/after | Replaces fake proof |
| Prompts | Customization prompts | Makes AI useful instead of generic |
| Checklist | Pre-use or pre-send checklist | Reduces mistakes |
| Planner | Sheet/worksheet | Turns content into an implementation tool |
| Compliance | Risk notes and safe wording | Avoids hype and unsafe claims |
| Funnel | FE, bump, OTO, license | Makes it sellable |
| Delivery | ZIP structure, README, Start Here | Makes it professional |

## Required Output For Any Product Creation Answer

1. Verdict: is the idea weak, ok, or strong?
2. Repositioned product name.
3. Buyer and pain.
4. Safe promise.
5. Mechanism.
6. FE product contents.
7. Use-case matrix.
8. Implementation workflow.
9. Examples to include.
10. Checklist.
11. Prompt/customization system.
12. Planner/sheet.
13. Compliance note.
14. Pricing, bump, OTO.
15. File/ZIP structure.
16. Today's build tasks.
