# AI PLR Rebrand Workflow / Campaign Map

## Purpose
This file turns raw PLR into a step-by-step rebrand and launch workflow. Do not sell unchanged PLR as the final product unless the license clearly allows it and the offer has been upgraded with original implementation assets.

## 7-Day Rebrand Workflow
| Day | Task | Output |
|---|---|---|
| 1 | Audit PLR source, niche, buyer, and license clues | PLR Evaluation Notes |
| 2 | Choose buyer, promise, and safe positioning | Offer Angle |
| 3 | Extract useful structure without copying wording | Rebuild Outline |
| 4 | Add original workflow, checklist, planner, and examples | Product Assets |
| 5 | Build sales page, FAQ, and objection blocks | Sales Assets |
| 6 | Build funnel, JV pack, WarriorPlus listing, and delivery page | Launch Assets |
| 7 | Run quality gate, license/risk review, export ZIP | Launch Pack |

## Campaign Map
| Campaign Asset | Job |
|---|---|
| Start Here | Tell buyer exactly what to open first |
| PLR Evaluation Checklist | Decide whether source material is usable |
| Rebrand Workflow | Convert source into original implementation kit |
| Offer Builder Template | Turn raw topic into buyer/promise/mechanism |
| Sales Page Template | Present the new offer clearly |
| JV Outreach Messages | Recruit affiliates without hype |
| WarriorPlus Launch Checklist | Prepare listing, delivery, support, and compliance |

## Quality Gate
- [ ] License/provenance reviewed or marked HUMAN REVIEW REQUIRED.
- [ ] No copied sales copy, swipe text, or income claim.
- [ ] New buyer avatar is clear.
- [ ] New product promise is realistic.
- [ ] At least one workflow, checklist, planner, and filled example exist.
- [ ] Sales page handles "Can I just ask AI?"
- [ ] Funnel and JV pack are present if launching on WarriorPlus.

## Decision Rule
If license is unclear, use the PLR as research only and sell the rebuilt original assets, not the original source files.
