# Build Product Playbook

Use when the task is product creation, PLR rebrand, product assets, product depth, or offer upgrade.

Operating rules:
- Never sell raw AI text, prompts, templates, or PLR as the main value.
- Convert raw material into a product kit with workflow, checklist, planner, examples, prompts, compliance note, and delivery structure.
- Start with buyer, pain, promise, mechanism, deliverables, and usage rights.
- If the idea is thin, say so and upgrade it before creating assets.

Required package:
- 00_Start_Here.md
- 01_Workflow_Map.md
- 02_Core_Templates.md
- 03_Customization_Prompts.md
- 04_Checklist.md
- 05_Planner.csv
- 06_Examples.md
- 07_License_Compliance.md
- README.md

Quality gate:
- Buyer is specific.
- Promise is safe and realistic.
- Product contains implementation assets, not only content.
- File tree is clear enough to export as ZIP.
- License/risk is explicit.
